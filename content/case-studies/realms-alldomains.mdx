# Case Study: Integrating AllDomains Identity into Realms via a Gibwork GitHub Bounty

<h2 className="text-3xl font-bold mt-8 mb-4">Executive Summary</h2>
All Domains, the Web3 identity layer for Solana domains, used Gibwork’s decentralized work marketplace to fund and manage the integration of its domain resolution system into the Realms governance platform. By posting a clear GitHub bounty on Gibwork, they attracted a skilled contributor who implemented the integration through an approved pull request.

The result: Realms users can now use AllDomains usernames to look up addresses, invite members, and display domain ownership, improving UX and deepening interoperability across Solana’s DAO tooling.

<h2 className="text-3xl font-bold mt-8 mb-4">About the Client</h2>
[All Domains](https://alldomains.id) is building the Web3 identity asset layer for Solana. Users can register personal blockchain-based domains, earn points, and manage their on-chain identity.

All Domains aims to make Solana more user-friendly and human-readable by turning wallet addresses into simple, meaningful domain names.

<h2 className="text-3xl font-bold mt-8 mb-4">The Challenge</h2>
All Domains wanted to increase adoption and utility of its domains by integrating them into [Realms](https://realms.today), the leading governance platform for Solana DAOs.

They needed contributors to modify the open-source Realms UI to:
- Let users enter an AllDomains username when inviting DAO members
- Resolve AllDomains usernames to wallet addresses in proposals
- Display users’ domains on DAO member pages
- Show domains owned by the DAO in the treasury
- Support domain transfers and domain detail views

In short: make AllDomains a first-class identity option in Realms.

This wasn’t a bug fix, it was a new feature integration across multiple Realms UI flows.

<h2 className="text-3xl font-bold mt-8 mb-4">The Gibwork Solution</h2>
All Domains chose Gibwork’s on-chain work marketplace to post the task as an open-source bounty linked directly to its GitHub repo.

Key steps:
- Created a clear GitHub issue with detailed requirements
> Issue #45: Integrate AllDomains into Realms [#45](https://github.com/Mythic-Project/governance-ui/issues/45)

- Posted the bounty on Gibwork offering $500 USDC Bounty
> Specified deliverables: new UI components, SDK integration, address resolution, domain listings, and links to AllDomains developer docs [Bounty](https://app.gib.work/bounties/ebeb8482-88f1-4a9d-8498-a147208fc1c4)

- Opened the task to global Web3 native developers
> Anyone with a wallet and connected GitHub account to Gibwork platform could participate, and submit.

- Managed work and acceptance transparently
> Pull request reviewed, tested, and merged

<h2 className="text-3xl font-bold mt-8 mb-4">Results</h2>
The bounty was picked up and completed by worker [Aryan](https://app.gib.work/p/0xaryan)

- Integration delivered in Pull Request [#49](https://github.com/Mythic-Project/governance-ui/pull/49)
- Code merged into the Realms UI main branch
- AllDomains resolution and display integrated across Realms flows

Realms users can now:
> Invite DAO members using AllDomains usernames
> View user domains on member pages
> Propose actions with AllDomains-based addressing
> See, transfer, and visit DAO-owned domains

This directly improved Realms’ UX and expanded AllDomains’ ecosystem integration.

<h2 className="text-3xl font-bold mt-8 mb-4">Conclusion</h2>
By posting a feature integration bounty on GibWork, All Domains funded critical Realms UI work without centralized hiring friction.

The task was:
- Clearly scoped via GitHub
- Open to the global developer community
- Paid securely in crypto on Solana
- Delivered via a merge-ready PR

This model demonstrates how Web3 projects can rapidly scale integrations and support open-source collaboration while proving contributor reputation and payment on-chain.
